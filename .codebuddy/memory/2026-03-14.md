## 游戏太简单 - 经验系统数值重平衡

### 问题分析
怪物经验掉落过多，核心原因是三层叠加效应：
1. **XP球掉落数 = ceil(怪物xp/2)**，怪物xp只决定球数，球值由`2+level*0.5`独立计算 → 双重乘法膨胀
2. **精英怪xp×5** → 球数暴增（精英豺狼人15xp → 8球）
3. **升级需求曲线太平缓**（8*lv^1.65 + lv*4），而经验收入远超需求
4. **升级回复20%HP** + 频繁升级 = 几乎不死

## 局内升级BUFF系统 — 增加策略性

### 设计思路
升级时不再只有技能选择，还混入了"增益BUFF"选项卡，让玩家可以选择不同的build方向（经济滚雪球、暴力输出、坦克防御、辅助机制），增加每局游戏的策略多样性。

### 实现内容

#### 1. BUFF数据库 (data.js BUFF_DB)
20个可叠加BUFF，分4大类：
- **经济型**: 求知欲(经验+20%)、点金术(金币+25%)、悟性(经验球+35%)、磁力场(拾取范围+30%)
- **攻击型**: 狂战之力(攻击+12%)、鹰眼(暴击+5%)、致命打击(暴伤+20%)、疾风(攻速+10%)、奥术增幅(技能伤害+15%)、屠杀者(精英/BOSS伤害+20%)

## 音效频率优化（解决短CD/被动技能音效轰炸）

## 爽快感+难度平衡大改（战力差距体现为实际难度）

### 问题
- 英雄战力远低于recPower（如10倍差距），但游戏体验毫无难度
- 升级太慢（1球=1xp, Lv2需18xp, 杀18只怪才升级），过程无聊
- 每次升级选择少（3选1），策略深度不足
- 怪物波次缩放太弱（线性15%/波），后期没有压力

### 修复

#### 爽快感提升
- **XP球价值**: BASE 1→3, SCALE 0.3→0.8（Lv1球值3, Lv5球值7）
- **XP球掉落**: 普通1→2球, 精英2→3球, BOSS 3→5球
- **升级经验需求降低**: XP_BASE 12→8, XP_POWER 1.85→1.55, XP_LINEAR 6→4
- **每级成长翻倍**: ATK+3/级(原1.8), HP+10/级(原5), 升级回血20%(原12%)
- **暴击频率**: 每3级+2%暴击(原5级+1%)
- **技能选择**: 默认4选1(原3选1), 刷新5选1(原4选1)
- **BUFF上限**: 面板最多3个BUFF(原2个)

#### 难度跟上
- **波次缩放**: HP 25%/波(原15%), ATK 18%/波(原10%), 速度3%/波(原2%)
- **指数因子**: 第5波起叠加1.08^(wave-4)指数增长（之前纯线性）
- **精英怪**: HP×5(原4), ATK×2.5(原2), 概率起步3%(原0%), 增长4%/波(原2.5%)
- **精英XP**: ×5(原3)，奖励挑战精英
- **BOSS缩放**: 12%/波(原5%)
- **战力差距倍率**: `powerGapMult = max(1, (recPower/heroPower)^0.4)`
  - 10倍差距→怪物2.51倍强度, 应用于小怪+BOSS的HP和ATK
- **刷怪压力**: 后期最大怪物数增长更快(*4→*3), 每批次增加更快, 刷怪间隔下限0.20s(原0.25)
- **无尽模式**: 指数缩放1.12→1.15/波
