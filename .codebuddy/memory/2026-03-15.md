## 修复NaN伤害 + 怪物血空不死亡bug (data.js + game.js)

### 根因
30个职业专属技能(3907-4281行)使用了错误调用：`calcSkillDmg(sk,l)+effectiveAtk*sk.atkRatio`（只传2参数，heroAtk=undefined→NaN）。正确应为 `calcSkillDmg(sk,l,effectiveAtk)`（传3参数）。NaN伤害导致 e.hp=NaN，NaN<=0为false，killEnemy永不触发。

### 修复
1. data.js: calcSkillDmg中 heroAtk 加默认值 `(heroAtk||0)`
2. game.js: 19处错误调用全部修复为正确的3参数形式
3. game.js: dmgEnemy函数添加多层NaN防护（入口+扣血前+hp修复）
4. game.js: 敌人主循环添加NaN hp检查，立即killEnemy清理

## 修复两个技能系统Bug (game.js)

### Bug 1: 技能升级显示DPS下降
- **根因**: `showSkillPickFeedback` 将升级前的"实际历史DPS"(`dmgStats.skillDmg/gameTime`)与升级后的"理论DPS"(`damage/cd`)对比。实际DPS受暴击/BUFF/目标密度影响，经常大于或小于理论值，导致升级后理论DPS反而"低于"实际DPS
- **修复**: 改为"升级前理论DPS vs 升级后理论DPS"的同口径对比。`prevDpsTheory = calcSkillDmg(sk,prevLv,S.attack) / calcSkillCd(sk,prevLv)`

### Bug 2: 技能升级到某等级时所有技能CD消失
- **根因**: `checkCombos()` 每次重建 `S.comboSkills` 列表。新解锁的合成技 `skillTimers` 为 undefined → 0 → 立即触发。对于萨满的"升腾(ascendance)"合成技，触发时执行 `setInterval` 将所有 `skillTimers > 0.5` 重置为 `0.1`，导致所有技能CD消失
- **修复**: `checkCombos` 中检测新增的合成技ID，为其初始化 `skillTimers[c.id] = c.cd` 为完整CD值，防止立即触发。同时用 `Set` 记录前一帧的合成技列表用于比对

  - 碰撞半径仅1.2，过小
  - 帧间隔硬编码0.016而非实际dt
- **修复**: 改用 `fireProjectile()` 统一投射物系统

  - 使用游戏主循环的真实dt
  - 碰撞半径为1.2(普通怪)/2.5(BOSS)
  - 保留火焰拖尾+爆炸AOE(2.5半径)+额外50%暴击率

### 优化: 治疗之雨视觉效果重做

  - 顶部5个Sprite组成的云雾层，缓慢飘动

### Bug修复：游戏中Live2D动效不生效
**根因**: CSS `filter: drop-shadow(...)` 应用在 `mm-hero-model` 容器上，Canvas每帧渲染时都会触发浏览器重新计算drop-shadow，性能极度消耗导致动画帧率降到几乎为0。此外容器缺少 `position:relative` 和固定尺寸，`refreshMainMenu` 每次调用都销毁重建实例。

**修复方案**:
1. `style.css`: 移除mm-hero-model的`filter: drop-shadow()`，添加`position:relative; width:280px; height:280px`
2. `systems.js`: 增加英雄ID缓存`_heroLive2DId`，只在英雄切换时重建Live2D实例
3. `hero-live2d.js`: `_setupCanvas`中确保容器设置`position:relative`和`overflow:hidden`

### Bug修复：file://协议下Live2D动效完全不工作
**根因**: `_loadImage()`中设置了`this.img.crossOrigin = 'anonymous'`，在`file://`协议下浏览器会因CORS策略拒绝加载本地图片，直接触发`onerror`回退到静态背景图。而`http://localhost`下同源请求无此问题。

**修复方案**: `hero-live2d.js` `_loadImage()`中，只在`location.protocol !== 'file:'`时设置`crossOrigin`，并在`onerror`中添加去掉`crossOrigin`后重试的逻辑。

1. **呼吸优化**：amplitude 从 0.010~0.012 降到 0.004，去掉X方向缩放（只保留Y微缩），避免腰部扭曲感
2. **飘动优化**：sway amplitude 降低约50%~60%，改用2D网格切片（SLICE_X=4, SLICE_Y=60）替代纯横条，让飘动只影响边缘区域
3. **战士剑光特效** `swordGleam`：剑身流动光斑 + 剑尖十字星芒 + 每4秒斩击弧光
4. **法师法杖光环** `staffAura`：旋转椭圆虚线环（3层）+ 环上亮点 + 6条能量射线 + 次要光源光环
5. **萨满闪电弧线** `lightning`：4条程序化锯齿闪电弧 + 三层渲染（光晕/主体/白线核心）+ 节点亮点 + 闪烁明灭

## 修正战士剑光特效坐标 (hero-live2d.js)

### 问题
战士形象上的剑光位置与实际图片中剑的位置不匹配。原坐标假设剑在画面中部偏右，但实际图片中矮人战士的大剑从左手(x≈0.28)握柄横跨到右上方(x≈0.80)剑尖。

### 修正内容
- 剑柄 `line.start`: [0.50, 0.28] → [0.28, 0.28]
- 剑尖 `line.end`: [0.62, 0.02] → [0.80, 0.04]
- 剑尖星芒 `tipStar.pos`: [0.62, 0.02] → [0.80, 0.04]
- 斩击弧光 `slashArc.origin`: [0.56, 0.15] → [0.55, 0.16]，半径0.18→0.22
- 剑身反光 glow `center`: [0.58, 0.08] → [0.55, 0.15]，半径0.15→0.18

### 验证
- hero-live2d.js lint: 0错误
- preview-live2d.html 预览确认

## 战士剑光遮挡衰减 + 速度优化 (hero-live2d.js)

### 问题
1. 剑在人脸后面，但剑光在人脸前面（穿透遮挡）
2. 光斑移动速度太快
3. 护手部分的高光没有停留

### 修改内容
1. **遮挡区域衰减**: 新增`occlusionZones`配置（头部[0.32,0.08,0.58,0.27] 92%遮挡 + 胡须[0.30,0.27,0.55,0.35] 60%遮挡），`_drawSwordGleam`改为分段绘制(20段基线+12段光斑)，每段检测遮挡可见度
2. **光斑速度降低**: `gleamSpeed` 1.8→0.6
3. **护手停留**: 新增`dwellZone`，在0~18%区间用ease-in-out减速到25%
4. **反光glow重分布**: 单一中心[0.55,0.15]拆为剑尖段[0.72,0.06]+护手段[0.30,0.26]，避开头部遮挡区
